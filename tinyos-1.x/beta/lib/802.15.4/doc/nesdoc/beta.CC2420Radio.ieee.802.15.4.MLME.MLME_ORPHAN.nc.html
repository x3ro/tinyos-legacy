Interface: MLME_ORPHAN

The MLME-SAP orphan notification primitives define how a coordinator can issue a notification of an orphaned device.
Author: Joe Polastre

Commands

  • void response(uint64_t OrphanAddress, uint16_t ShortAddress, bool AssociatedMember, bool SecurityEnable) Allows the next higher layer of a coordinator to respond to the indication primitive

    Parameters:

    OrphanAddress - The 64-bit extended address of the orphaned device

    ShortAddress - The short address allocated to the orphaned device if it is associated with this coordinator

    AssociatedMember - TRUE if the orphaned device is associated with this coordinator

    SecurityEnable - TRUE if security is enabled for this transfer

Events

  • void indication(uint64_t OrphanAddress, bool SecurityUse, uint8_t ACLEntry) Allows the MLME of a coordinator to notify the next higher layer of the presence of an orphaned device

    Parameters:

    OrphanAddress - The 64-bit extended address of the orphaned device

    SecurityUse - TRUE if security is enabled for this transfer

    ACLEntry - The macSecurityMode parameter value from the ACL entry associated with the sender of the data frame
