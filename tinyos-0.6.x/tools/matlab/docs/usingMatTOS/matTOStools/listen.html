Listen

Listen will print all packets from a particular set of motes or from a particular set of serial ports.  To start listen, make sure that the motes you want to listen to are mapped to a packetPort in your environment.  Then, to listen to motes 1, 5, 101 and 34,just type

listen([1 5 101 34])

or, if you want to listen to all packets on a particular port, for example COM1 and COM2,type

listen(getPacketPorts({'COM1', 'COM2'}))

or if you want to listen to all ports and all motes:

listen('all')

What if you don't want to read hexidecimal byte streams?  You want decimal byte streams instead?  Change the display mode with this command:

setListenDisplayMode('decimal')

or just 'd' for short.  What if you want to see the real numbers in the packet, and the real structure?

setListenDisplayMode('verbose')

What if you are getting somebody else's packets that clutter your screen (a very common problem)?  Or what if you only want to listen to packets of a certain AM type.  You can filter all packets that don't come from a certain set of motes, have a certain AM type or group ID using the following three commands:

setListenAMfilter(8)
setListenMoteIDfilter([1 2 3 4 100 101 102])
setListenGroupIDfilter(hex2dec('7d'))

You can also do all of this as command-line arguments as follows:

listen([1 2 3 4 100 101 102], 'verbose', 8, hex2dec('7d'))
