MatTOS

The purpose of matTOS is to provide an easy way to prototype applications and analyze sensor data.  This environment has a few advantages over the Java tool set, including:

  • An interactive, interpreted environment
  • Powerful data analysis and graphing

It also has disadvantages, including the fact that it is not open-source and it is slower. 

Since matTOS is being built from scratch, there were a few things we included to make it much more useful, but these can and should also be done for the java tools environment also.

  1. An entire communications stack has been implemented on the PC side.  This allows you to deal with several different abstractions of your data.
  2. A general packet object was created to allow you to get completely away from bytes and just think about your data.
  3. Many standard utilities and tools have been abstracted and standardized for easy code sharing.
  4. A directory structure that mirrors that of the TOS distribution is used for new tools, allowing us to keep specific modifications of code in our own directory while leaving general tools to be used by everybody.  It also emphasizes that we need a new tool to deal with each TOS component
  5. Simple manipulation of multiple serial ports
