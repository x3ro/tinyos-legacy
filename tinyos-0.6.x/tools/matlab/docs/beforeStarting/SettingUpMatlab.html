Setting Up Matlab

 

Setting up matlab is pretty straightforward. Make sure that you install the help files; you’ll be sorry if you don’t.  You also probably want to install most of the signal processing, math, and statistics toolboxes.

  

In order to have the matlab serial port interface, you need to have at least Matlab Release 12.0.  Currently, matTos is compatible with matlab R12, R12.1 and R12.5.  If you use the old commStack in MatTos you will have compatibility issues with 12.1, but see the 6.1 commStack for more details.

 

To use the new commStack, you must type "edit classpath.txt" in your matlab command window and add the line "C:/nest/tools" or whatever the path to your nest/tools/ directory is.  Then, restart matlab.

 

If you are running winXP, matlab Release 12.0 will require some fixes.  (If you have a Pentium 4 processor, Matlab R12.0 might not start up at al.  The following will fix that as well.)  The fix is the following:

    1.  go to the “matlabR12\sys\java\jre\win32\” directory and copy the old “jre” directory to “jreOld”.   

    2.  copy the “jre” directory from a java jdk (I used jdk1.3.1_01, which comes with the tinyOS distribution CD) in that directory. 

    3.  Copy the “swingall.jar” file from the “jreOld/lib” to “jre/lib”. 

    4.  Now it should work.  You might get funny java exceptions about your clipboard when you open matlab, but just copy and paste something and the exceptions stop. 

 

If you are using windows2000 you may have problems using the COMM port redirector, which is used to connect to sockets via the matlab serial port interface.  This is only important if you are trying to access serial ports remotely.
