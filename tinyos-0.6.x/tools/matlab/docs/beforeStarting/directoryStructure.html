directory structure

All matlab code is held in the "nest/tools/matlab" directory.  Inside are a few directories.  Things in green have not been implemented yet.  The heart of matTOS is the commStack and the packet object.  The apps and shared directories are for commands that directly correspond to things in the /nest/apps and /nest/tos/shared directories.  The tools directory holds things that are general enough to be used by everybody but have to do only with matlab and have nothing to do with TOS.  The contrib directory holds things that have to do only with matlab and have nothing to do with TOS, but are also not general enough for everybody to use.  Every individual should start off by making themselves a contrib directory.  Then, as their work becomes generalizable, they should move it into the tools directory.

  1. @packet:                                The packet object and related functions.
  2. commStack:                            The PC-side communications stack
    1. dataLayer:                  The data interface on the communications stack
      1. Serial:              The interface to physical serial ports
      2. Socket:            The interface to network sockets
      3. USB:               The interface to physical USB ports
    2. packetLayer               the packet interface on the communications stack
    3. AMlayer:                    the active message interface on the comm. Stack
    4. routingLayer              the routing interface on the comm. Stack
  3. contrib:                                   work by one person that is not necessarily general enough to be used by others
  4. apps:                                        Tools for the components in the “tos/apps” directory
  5. shared:                                    Tools for the components in the “tos/shared” directory
  6. tools:                                       General user-interface tools that should be shared
    1. listen:                          Prints streaming packets on a packetPort
    2. watch:                          Graphically displays streaming data on a packetPort.
    3. watchSurface:               Graphically displays a surface plot of streaming data on a packetPort
  7. util:                                          common utilities such as hex-dec conversion
  8. docs:                                        documentation.  Specific apps might have their own docs in their own directories.
  9. demos                                     Matlab tool demos
